# Clipboard History App - Quick Reference Guide

## Platform APIs Quick Reference

### macOS (NSPasteboard)

```swift
// Monitor clipboard
let pasteboard = NSPasteboard.general
var lastChangeCount = pasteboard.changeCount

Timer.scheduledTimer(withTimeInterval: 0.5, repeats: true) { _ in
    if pasteboard.changeCount != lastChangeCount {
        lastChangeCount = pasteboard.changeCount
        // Process clipboard change
    }
}

// Read clipboard
if let string = pasteboard.string(forType: .string) {
    print("Text: \(string)")
}

// Write to clipboard
pasteboard.clearContents()
pasteboard.setString("Hello", forType: .string)

// Ignore sensitive types
let ignoredTypes: Set<NSPasteboard.PasteboardType> = [
    .init("org.nspasteboard.ConcealedType"),
    .init("org.nspasteboard.TransientType"),
    .init("org.nspasteboard.AutoGeneratedType")
]
```

### Windows (Win32 API)

```cpp
// Monitor clipboard (modern approach)
AddClipboardFormatListener(hWnd);

// In WndProc
case WM_CLIPBOARDUPDATE:
    HandleClipboardChange();
    break;

// Read clipboard
if (OpenClipboard(hWnd)) {
    HANDLE hData = GetClipboardData(CF_UNICODETEXT);
    if (hData) {
        wchar_t* text = (wchar_t*)GlobalLock(hData);
        // Use text
        GlobalUnlock(hData);
    }
    CloseClipboard();
}

// Write to clipboard
if (OpenClipboard(hWnd)) {
    EmptyClipboard();
    HGLOBAL hMem = GlobalAlloc(GMEM_MOVEABLE, size);
    // Copy data to hMem
    SetClipboardData(CF_UNICODETEXT, hMem);
    CloseClipboard();
}

// Ignore flag
UINT cfIgnore = RegisterClipboardFormat(TEXT("Clipboard Viewer Ignore"));
if (IsClipboardFormatAvailable(cfIgnore)) {
    // Don't save
}
```

### Linux (Qt Cross-Platform)

```cpp
// Monitor clipboard
QClipboard* clipboard = QApplication::clipboard();
connect(clipboard, &QClipboard::dataChanged, this, &MyClass::onClipboardChange);

// Read clipboard
QString text = clipboard->text();
QImage image = clipboard->image();

// Write to clipboard
clipboard->setText("Hello");
clipboard->setImage(image);
```

## Database Schema

```sql
-- Main clipboard items table
CREATE TABLE clipboard_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    content_type TEXT NOT NULL,
    content_text TEXT,
    content_blob BLOB,
    source_app TEXT,
    copied_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_used_at TIMESTAMP,
    use_count INTEGER DEFAULT 1,
    is_pinned BOOLEAN DEFAULT 0
);

-- Multiple formats per item
CREATE TABLE clipboard_formats (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    item_id INTEGER,
    format_name TEXT,
    format_data BLOB,
    FOREIGN KEY (item_id) REFERENCES clipboard_items(id) ON DELETE CASCADE
);

-- Indexes for performance
CREATE INDEX idx_copied_at ON clipboard_items(copied_at DESC);
CREATE INDEX idx_content_type ON clipboard_items(content_type);
CREATE INDEX idx_source_app ON clipboard_items(source_app);
```

## Security Checklist

- [ ] Ignore `ConcealedType` (macOS)
- [ ] Ignore `TransientType` (macOS)
- [ ] Ignore `AutoGeneratedType` (macOS)
- [ ] Check for "Clipboard Viewer Ignore" format (Windows)
- [ ] Allow application ignore list
- [ ] Implement content filtering (regex)
- [ ] Encrypt database (optional, AES-256)
- [ ] Configurable data retention
- [ ] Clear history option
- [ ] Request necessary permissions

## Performance Best Practices

1. **Monitoring Interval:** 200-500ms for polling (macOS)
2. **Use Events:** `AddClipboardFormatListener` on Windows
3. **Background Processing:** Database writes on background thread
4. **Lazy Loading:** Load items on demand
5. **Pagination:** Display 50-100 items at a time
6. **Indexing:** Index timestamp and content type columns
7. **Prepared Statements:** Use for frequent queries
8. **Memory Management:** Don't load entire history at once

## Common Pitfalls

1. **Not ignoring sensitive data** - Always filter password manager content
2. **Polling too frequently** - Wastes CPU, 500ms is usually fine
3. **Blocking UI thread** - Use background threads for database operations
4. **Not handling large images** - Compress or store thumbnails
5. **Forgetting permissions** - Request Accessibility on macOS
6. **Poor search performance** - Index database properly
7. **Memory leaks** - Clean up old clipboard items
8. **Cross-platform clipboard differences** - Test on all target platforms
