# Clipboard History App - Implementation Summary for Claude AI CLI

This document provides a comprehensive summary of the technical research on building a clipboard history application, specifically designed to provide maximum context for Claude AI CLI.

## Executive Summary

A clipboard history app is a utility that monitors the system clipboard and stores a history of copied items, allowing users to access and paste previously copied content. This research covers the implementation details of popular solutions like Alfred, Maccy, Ditto, and CopyQ, along with platform-specific clipboard APIs for macOS, Windows, and Linux.

## Key Open Source Projects

### 1. Maccy (macOS)
- **Repository:** https://github.com/p0deje/Maccy
- **Language:** Swift
- **Architecture:** Native macOS app using SwiftUI/AppKit
- **Clipboard Monitoring:** Timer-based polling of `NSPasteboard.general.changeCount` (default 500ms interval)
- **Data Storage:** SwiftData (SQLite) at `~/Library/Application Support/Maccy/Storage.sqlite`
- **Key Features:**
  - Ignores sensitive types: `org.nspasteboard.ConcealedType`, `org.nspasteboard.TransientType`, `org.nspasteboard.AutoGeneratedType`
  - Application-specific filtering (ignore/whitelist)
  - Regular expression filtering for content
  - Keyboard-first UI with customizable shortcuts
  - Pin items, search history
  
**Code Example - Clipboard Monitoring:**
```swift
func start() {
  timer = Timer.scheduledTimer(
    timeInterval: Defaults[.clipboardCheckInterval],
    target: self,
    selector: #selector(checkForChangesInPasteboard),
    userInfo: nil,
    repeats: true
  )
}

@MainActor
func checkForChangesInPasteboard() {
  guard pasteboard.changeCount != changeCount else {
    return
  }
  changeCount = pasteboard.changeCount
  // Process new clipboard content
}
```

- **Clipboard Monitoring:** Uses `SetClipboardViewer()` and `AddClipboardFormatListener()` for change notifications
- **Data Storage:** SQLite database (`Ditto.db`)
- **Key Features:**

  - Encrypted network transfers

## Platform-Specific Clipboard APIs

### macOS: NSPasteboard

**Key Concepts:**
- `NSPasteboard.general` - The system clipboard
- `changeCount` - Increments on every clipboard change (polling required)
- No push notifications - must poll periodically
- Multiple data types per clipboard item
- Requires Accessibility permissions for full functionality

**Important Pasteboard Types to Ignore:**
- `org.nspasteboard.ConcealedType` - Sensitive data (passwords)
- `org.nspasteboard.TransientType` - Temporary data
- `org.nspasteboard.AutoGeneratedType` - Auto-generated content
- `source` attribute - Indicates Universal Clipboard items

**Supported Types:**
- `NSPasteboard.PasteboardType.string`
- `NSPasteboard.PasteboardType.html`
- `NSPasteboard.PasteboardType.rtf`
- `NSPasteboard.PasteboardType.png`
- `NSPasteboard.PasteboardType.tiff`
- `NSPasteboard.PasteboardType.fileURL`

**Reference:** http://nspasteboard.org/ - Essential guidelines for privacy-respecting clipboard managers

### Windows: Win32 Clipboard API

**Key Functions:**
- `AddClipboardFormatListener(hWnd)` - Modern change notification (recommended)
- `OpenClipboard(hWnd)` - Open clipboard for access
- `GetClipboardData(format)` - Retrieve data in specific format
- `SetClipboardData(format, hMem)` - Place data on clipboard
- `EmptyClipboard()` - Clear clipboard
- `CloseClipboard()` - Close clipboard handle
- `RegisterClipboardFormat(name)` - Register custom format

**Clipboard Formats:**
- `CF_TEXT` - Plain text (ANSI)
- `CF_UNICODETEXT` - Unicode text
- `CF_BITMAP` - Bitmap image
- `CF_HDROP` - File list
- `CF_HTML` - HTML content
- Custom formats via `RegisterClipboardFormat()`

**Change Notification Pattern:**
```cpp
// In window initialization
AddClipboardFormatListener(hWnd);

// In window procedure
case WM_CLIPBOARDUPDATE:
    // Clipboard has changed
    HandleClipboardChange();
    break;
```

### Linux: X11 and Wayland

**X11 Selections:**
- `PRIMARY` - Middle-click paste (selected text)
- `CLIPBOARD` - Ctrl+C/Ctrl+V operations
- `SECONDARY` - Rarely used

**Key Challenge:** Data is stored in the source application's memory. If the app closes, clipboard data is lost.

**X11 Tools:**
- `xclip` - Command-line clipboard access
- `xsel` - Alternative clipboard tool

**Wayland:**
- Compositor-mediated clipboard
- Similar data persistence issue
- `wl-clipboard` - Wayland clipboard tools
- `wl-clip-persist` - Daemon to persist clipboard data

**Cross-Platform Solution:**
Use Qt framework (`QClipboard`) or implement platform-specific backends with a unified interface.

## Data Storage Strategies

### SQLite Database Schema (Recommended)

```sql
CREATE TABLE clipboard_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    content_type TEXT NOT NULL,  -- 'text', 'image', 'file', 'html', 'rtf'
    content_text TEXT,
    content_blob BLOB,
    source_app TEXT,
    copied_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_used_at TIMESTAMP,
    use_count INTEGER DEFAULT 1,
    is_pinned BOOLEAN DEFAULT 0,
    is_favorite BOOLEAN DEFAULT 0
);

CREATE TABLE clipboard_formats (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    item_id INTEGER,
    format_name TEXT,
    format_data BLOB,
    FOREIGN KEY (item_id) REFERENCES clipboard_items(id) ON DELETE CASCADE
);

CREATE INDEX idx_copied_at ON clipboard_items(copied_at DESC);
CREATE INDEX idx_content_type ON clipboard_items(content_type);
```

### Encryption

**Recommended:** AES-256-GCM for database encryption

**Implementation Options:**
1. **SQLCipher** - Encrypted SQLite extension
2. **Application-level encryption** - Encrypt sensitive fields before storage
3. **Full database encryption** - Encrypt entire database file

**Performance Consideration:** Encryption adds overhead. Consider encrypting only sensitive data or providing it as an optional feature.

## Performance Optimization

### 1. Clipboard Monitoring

**macOS:**
- Polling interval: 200-500ms (balance between responsiveness and CPU usage)
- Use `DispatchQueue` for background monitoring

**Windows:**
- Use `AddClipboardFormatListener` (event-driven, more efficient than polling)

**Linux:**
- Use Qt's signal/slot mechanism for clipboard changes

### 2. Database Operations

- Use prepared statements for frequent queries
- Batch inserts for multiple items
- Index frequently queried columns (timestamp, content type)
- Use transactions for multiple operations

### 3. Memory Management

- Lazy load clipboard items (don't load all at once)
- Implement pagination for large histories
- Cache frequently accessed items
- Compress large text items
- Store file paths instead of file contents for file clipboard items

### 4. Background Processing

- Use background threads/queues for:
  - Database writes
  - Image thumbnail generation
  - Content indexing for search
- Keep UI thread responsive

## Code Examples and Patterns

### macOS: Complete Clipboard Monitor

```swift
import AppKit
import SwiftData

class ClipboardMonitor {
    private let pasteboard = NSPasteboard.general
    private var changeCount: Int
    private var timer: Timer?
    private let modelContext: ModelContext
    
    init(modelContext: ModelContext) {
        self.modelContext = modelContext
        self.changeCount = pasteboard.changeCount
    }
    
    func start(interval: TimeInterval = 0.5) {
        timer = Timer.scheduledTimer(
            withTimeInterval: interval,
            repeats: true
        ) { [weak self] _ in
            self?.checkClipboard()
        }
    }
    
    func stop() {
        timer?.invalidate()
        timer = nil
    }
    
    private func checkClipboard() {
        guard pasteboard.changeCount != changeCount else { return }
        changeCount = pasteboard.changeCount
        
        guard let items = pasteboard.pasteboardItems else { return }
        
        for item in items {
            if shouldIgnore(item) { continue }
            saveClipboardItem(item)
        }
    }
    
    private func shouldIgnore(_ item: NSPasteboardItem) -> Bool {
        let ignoredTypes: Set<NSPasteboard.PasteboardType> = [
            .init("org.nspasteboard.ConcealedType"),
            .init("org.nspasteboard.TransientType"),
            .init("org.nspasteboard.AutoGeneratedType")
        ]
        
        return item.types.contains { ignoredTypes.contains($0) }
    }
    
    private func saveClipboardItem(_ item: NSPasteboardItem) {
        let historyItem = HistoryItem()
        
        // Get source application
        if let bundleID = NSWorkspace.shared.frontmostApplication?.bundleIdentifier {
            historyItem.application = bundleID
        }
        
        // Save different content types
        if let string = item.string(forType: .string) {
            let content = HistoryItemContent(type: "string", value: string)
            historyItem.contents.append(content)
        }
        
        if let rtf = item.data(forType: .rtf) {
            let content = HistoryItemContent(type: "rtf", value: rtf.base64EncodedString())
            historyItem.contents.append(content)
        }
        
        // Add more types as needed...
        
        modelContext.insert(historyItem)
        try? modelContext.save()
    }
}
```

### Windows: Clipboard Listener

```cpp
class ClipboardMonitor {
private:
    HWND m_hWnd;
    
public:
    ClipboardMonitor(HWND hWnd) : m_hWnd(hWnd) {
        AddClipboardFormatListener(m_hWnd);
    }
    
    ~ClipboardMonitor() {
        RemoveClipboardFormatListener(m_hWnd);
    }
    
    void HandleClipboardUpdate() {
        if (!OpenClipboard(m_hWnd)) return;
        
        // Check for ignore format
        UINT cfIgnore = RegisterClipboardFormat(TEXT("Clipboard Viewer Ignore"));
        if (IsClipboardFormatAvailable(cfIgnore)) {
            CloseClipboard();
            return;
        }
        
        // Get text
        if (IsClipboardFormatAvailable(CF_UNICODETEXT)) {
            HANDLE hData = GetClipboardData(CF_UNICODETEXT);
            if (hData) {
                wchar_t* text = static_cast<wchar_t*>(GlobalLock(hData));
                if (text) {
                    SaveToDatabase(text);
                    GlobalUnlock(hData);
                }
            }
        }
        
        CloseClipboard();
    }
    
    void SaveToDatabase(const wchar_t* text) {
        // Save to SQLite database
    }
};

// In window procedure
LRESULT CALLBACK WndProc(HWND hWnd, UINT message, WPARAM wParam, LPARAM lParam) {
    static ClipboardMonitor* monitor = nullptr;
    
    switch (message) {
    case WM_CREATE:
        monitor = new ClipboardMonitor(hWnd);
        break;
        
    case WM_CLIPBOARDUPDATE:
        if (monitor) monitor->HandleClipboardUpdate();
        break;
        
    case WM_DESTROY:
        delete monitor;
        PostQuitMessage(0);
        break;
        
    default:
        return DefWindowProc(hWnd, message, wParam, lParam);
    }
    return 0;
}
```

### Cross-Platform with Rust (Tauri)

```rust
use tauri::Manager;
use clipboard_master::{CallbackResult, ClipboardHandler, Master};
use std::sync::mpsc;

struct ClipboardMonitor {
    tx: mpsc::Sender<String>,
}

impl ClipboardHandler for ClipboardMonitor {
    fn on_clipboard_change(&mut self) -> CallbackResult {
        if let Ok(contents) = clipboard::get_text() {
            let _ = self.tx.send(contents);
        }
        CallbackResult::Next
    }
}

#[tauri::command]
fn start_clipboard_monitor(app_handle: tauri::AppHandle) {
    let (tx, rx) = mpsc::channel();
    
    // Start clipboard monitor in background thread
    std::thread::spawn(move || {
        let monitor = ClipboardMonitor { tx };
        let _ = Master::new(monitor).run();
    });
    
    // Handle clipboard changes
    std::thread::spawn(move || {
        while let Ok(text) = rx.recv() {
            app_handle.emit_all("clipboard-change", text).unwrap();
        }
    });
}
```

## Key Takeaways for Implementation

1. **Platform-Specific Monitoring:** Each OS requires different clipboard monitoring approaches. macOS needs polling, Windows can use event listeners, Linux requires handling X11/Wayland differences.

2. **Privacy First:** Always implement sensitive data filtering. Respect `ConcealedType`, `TransientType`, and password manager flags.

3. **SQLite for Storage:** Use SQLite for structured, efficient storage with proper indexing.

4. **Keyboard-Driven UI:** Users expect fast, keyboard-driven interaction with clipboard managers.

5. **Performance Matters:** Implement lazy loading, background processing, and efficient database queries to keep the app responsive.

6. **Security:** Consider encryption for sensitive users, but be aware of performance trade-offs.

7. **Open Source Reference:** Study Maccy (macOS), Ditto (Windows), and CopyQ (cross-platform) for real-world implementation patterns.
